# Описание изменений в monitorings/meat

## Новый функционал

### Обработка базы данных при запуске пайплайна

#### Функция update_empty_product_details()
- **Назначение**: Автоматическое заполнение пустых значений `product_details` в базе данных при первом запуске пайплайна
- **Логика работы**:
  - Поиск записей с пустыми `product_details` в таблице `raw_test_test`
  - Извлечение уникальных описаний продуктов
  - Анализ описаний с помощью регулярных выражений для выявления деталей продукта
  - Обновление БД с заполненными значениями
- **Ограничения**: Обрабатывает до 10,000 записей за раз для оптимизации производительности
- **Логирование**: Подробное отслеживание прогресса

#### Интеграция в пайплайн
- Вызов при первом запуске системы мониторинга
- Повторная проверка после обработки каждого файла

### Извлечение product_details из новых файлов

#### Функция extract_product_details()
- **Назначение**: Анализ описаний продуктов для автоматического извлечения деталей
- **Логика работы**: Использование регулярных выражений из `product_details_patterns`
- **Результат**: Список найденных деталей продукта (например: `['нк', 'с хребтом']`)

#### Интеграция в process_dataframe_cols()
- **Автоматическое извлечение**: При обработке каждого нового файла мониторинга
- **Оптимизация**: Обработка уникальных описаний для избежания дублирования вычислений
- **Сериализация**: Автоматическое преобразование списков в JSON-строки для корректного хранения в БД
- **Fallback**: Если детали не найдены, устанавливается значение `["Не обнаружено"]`

## Техническая реализация

### Паттерны извлечения
- Используются предопределённые регулярные выражения в `product_details_patterns`
- Case-insensitive поиск для повышения точности

## Логирование и мониторинг

### Уровни логирования
- **INFO**: Прогресс обработки, количество записей, успешные операции
- **WARNING**: Повторная обработка файлов
- **ERROR**: Ошибки подключения к БД, сбои в обработке

## Использование

Функционал работает автоматически при:
1. Запуске пайплайна мониторинга
2. Обработке новых файлов мониторинга
3. Повторной обработке существующих файлов
