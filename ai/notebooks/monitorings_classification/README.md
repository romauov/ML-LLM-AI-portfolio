Цель исследования заключается в классификации данных мониторинга с помощью моделей, без использования ChatGPT.

### Разведочный анализ данных:

Ноутбук - [eda.ipynb](eda.ipynb)

На момент исследования у нас есть 3 категории (баранина, свинина и говядина) для которых уже написаны две модели и сделаны предсказания с помощью ChatGPT и 1 категория (птица) для которой у нас есть только свои модели. За самую точную модель берем ChatGPT.

Основная проблема заключается в том, что у нас данные не размечены, поэтому при обучении модели используется метод обучения без учителя. Немного информации о точности моделей.

Совпадения предсказаний моделей для баранины:
- Между собой 73%
- Между моделью 1 и ChatGPT 78%
- Между моделью 2 и ChatGPT 88%

Совпадения предсказаний моделей для говядины:
- Между собой 84%
- Между моделью 1 и ChatGPT 85%
- Между моделью 2 и ChatGPT 96%

Совпадения предсказаний моделей для свинины:
- Между собой 76%
- Между моделью 1 и ChatGPT 81%
- Между моделью 2 и ChatGPT 93%

Совпадения предсказаний моделей для птицы:
- Между собой 54%

### Простые алгоритмы

Ноутбук - [stemming_lemmatization_predictions.ipynb](stemming_lemmatization_predictions.ipynb)

Предсказания основаны на поиске вхождения названия категории в описании. Описание и категория предварительно обрабатываются от лишних символов и затем применяется процесс приведения слова к изначальной форме (лемме) или нахождения основы слова (стемминг).

### Классический ML

Ноутбук - [tfidf_truncated-svd.ipynb](tfidf_truncated-svd.ipynb)

Предсказания основанные на алгоритме TF-IDF. С помощью этого алгоритма получаются векторные представления слов (эмбеддинг). Затем эмбеддинги слов используются, как признаки в алгоритмах кластеризации (k-means, DBSCAN). Однако, такой подход показывает очень слабые результаты по сравнению с нахождением косинусного сходства эмбеддинга описания и эмбеддинга категории.

### DL

Ноутбук - [bert.ipynb](bert.ipynb)

Используются предобученные нейронные сети архитектуры Bert на корпусах текстов русского языка. Модели дополнительно предобучаются на описаниях из данных мониторинга. Затем с помощью косинусного сходства предсказывается категория. Такой подход позволяет использовать одну модель для предсказаний под разные категории данных (баранина, свинина, говядина и птица).

Используются две модели:
- rubert-tiny2 (115мб)
- distilrubert-base-cased-conversational (520мб)

Сходство модели rubert-tinny2 с ChatGPT:
- говядина 92.24%
- свинина 94.24%
- баранина 76.29%

Сходство модели distilrubert-base-cased-conversational с ChatGPT:
- говядина 89.57%
- свинина 94.61%
- баранина 91.42%
