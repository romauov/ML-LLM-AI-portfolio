# Описание работы Pipeline
Pipeline состоит из нескольких основных компонентов:

1. Подготовка данных:
    1. Объединение type1 и type2 в один продукт
    2. Приведение к нижнему регистру продукции
    3. Расчет взаимодействия пользователя с продуктом
    4. Построение сводной таблицы, пропуски которой заполняются нулями
2. Построение модели ближайшего соседа для идентификации 3 похожих продуктов
3. Фильтрация пользователей:
    1. По последней активности на сайте
    2. Удаление игнорщиков/спамеров
    3. Удаление пользователей, которые редко взаимодействуют с продукцией(type1)
4. Построение списков:
    1. Списки строятся на основе краткосрочных интересов, если количество найденных пользователей меньше целевого - увеличивается период поиска
    2. В первую очередь отбираются пользователи, которые работают с целевым продуктом(указанным в запросе), затем с продукцией, которую определила модель
    3. Если количество пользователей меньше целевого, производится отбор пользователей, которые работали с type1

Используемые данные:
1. userStat
2. userId, email из userProfile 
3. Статистика открытий из userStat, type in ('email_sent', 'email_open')

# Варианты ответов
## Штатная ситуация
Возвращается json({"emails": ["sherbakov-1990@list.ru", "sergei-076@mail.ru"], "userIds": [266253, 43263]})

## Количество пользователей меньше необходимого
Возвращается максимальное количество пользователей, оставшихся после фильтрации

## Продукт не найден
Если переданный продукта(type1 + type2) не удалось найти, то поиск пользователей будет осуществляться по type1. Например, было передано: "Говядина шоколадная", будет осуществляться поиск среди пользователей, которые работали с Говядиной

## Type1 не найден
В случае ненайденного type1 продукта, например, передано "Говяндина нарезка", сначала осуществится поиск по продукту "Говяндина нарезка", затем по "Говяндина". 

Вернется json({"error": "type1='Говяндина' не найден"})
