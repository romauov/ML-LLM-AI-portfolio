# Описание работы Pipeline

Принцип работы построен на поиске похожих объявлений, и пользователей которые с ними взаимодействуют. Похожие объявления ищутся по векторной схожести, векторы получены при помощи LLM.

Pipeline состоит из нескольких основных компонентов:

1. Подготовка данных:
    1. Создание одного df:
        1. Объедние(Merge) userstat и tradeboard по ID объяления
        2. Сортировка по дате
        3. Создание 'context' - объединение название и текста объявления(title, descr)
        4. Создание 'search' - объединение категорий товара (type1, type2)
    2. Список покупателей(
                покупатель - человек который ищет товар, смотрит объявления о продаже, создает объявления о покупке и т.д.)
    3. Список уникальных объявлений о продаже
    4. Список уникальных поисковых запросов
2. Поиск 10 похожих объявлений и поисковых запросов на основе схожести векторов
3. Фильтрация пользователей:
    1. По последней активности на сайте
    2. Удаление админов
    3. Удаление пользователей участвующих в рассылке
4. Построение списка

*5. (ОПЦИОНАЛЬНО) Если длина списка меньше необходимой то количество похожих объявлений и поисковых запросов увеличивается на 10 и список пересобирается*

*6. (ОПЦИОНАЛЬНО) Это повторяется 3 раза. Если список так и не собрается, то объявление рассылется 10 послденим активным пользователям*


Используемые данные:
1. userStat
2. tradeboard
3. embeddings for meatinfo and fishretail
