# Трекер времени 

Api доступно по url: (https://ai.m16.tech/api/tracker)

Демо страница доступна по url: (https://ai.m16.tech/gradio/tracker)

Api возвращает проект, тикет, затраченное время время, дату, вид работ, заметку

Способен работать с датами в следующих форматах: 
* Указанием вчера/сегодня
* Дни недели
* Даты в форматах 2 мая

# Обращение к api и получение результата

Api получает json в формате: {"text": "Сообщение"}

Результат представляет собой: 
    {'project': 'm16',
     'ticket': 0,
     'date': '28.05.2023',
     'spended_time': 45,
     'work_type': 'Communication',
     'text': 'стенд ап'}

## Значения по умолчанию

Если проект не найден в сообщении, будет возвращено 'project': '-'

При отсутствии тикета будет возвращен 0 - работа без тикета

При отсутствии даты будет возвращен сегодняшний день в формате 28.05.2023

При отсутствии затраченного времени будет возвращен 0

При отсутствии заметки, поле text будет пустым, work_type: '-'

# Обучение и переобучение модели

Для обучение модели можно использовать google colab: (https://colab.research.google.com/drive/1n1sycICJPk41RSlHOIsDbYm4euihvL3_?usp=sharing)

В случае недоступности ноутбука: 
Взять ноутбук tracker и загрузить его в colab
1. Скачать данные трекинга из (http://devstars.a505.ru/tracking), поставив 2021 год, чтобы все данные попали в выборку
2. Вставить файл test.csv в ноутбук
3. Нажать runtime -> change runtime type, выбрать GPU
4. Нажать runtime -> run all
5. Авторизоваться в гугл диске для последующего сохранения модели
6. Модель будет сохранена на гугл диске в папке model
7. Заменить файлы в apps/tracker/data/model
8. Отправить изменения модели
```
./bin/dvc add apps/tracker/data/model
git commit apps/tracker/data/model.dvc -m "Model updates"
./bin/dvc push
```
9. Сделать релиз

# Архитектура и датасет

Модель построена на основе двунапревленного LSTM. Одна ветвь отвечает за поиск именнованных сущностей, другая - определение вида работ

Датасет был построен на основе взятых данных из трекера, на основе алгоритма генерации (находится в jupyter ноутбуке - random_concatenate) было сгенерировано 300 000 образцов, 5% которых были отобраны для валидации
