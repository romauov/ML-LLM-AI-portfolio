Алгоритм формирования списков пользователей для рассылки

Основан на взвешивании и аггрегировании событий активных пользователей сайта по каждой категории мяса, вида разруба

На вход получает следующие параметры:
- ID axe'a - идентификатор акса, от имени которого производится рассылка
- Категории продукции. Допустимы 2 формата ввода:


    1. Если хотим подобрать пользователей, которые взаимодействовали со всеми подкатегориями указанных категорий
    Говядина, 
    Свинина

    2. Если хотим подобрать пользователей, которые взаимодействовали только с указанными подкатегориями. 
    Говядина: язык, шейка
    Свинина: окорок


- Период активности. Промежуток времени (в днях), в течение которого берутся события пользователей (NUMBER_OF_DAYS). Больше значение - больше пользователей.

- Дата последнего открытия рассылки пользователем (LAST_MAIL_OPENING_DATE). Более ранняя дата - больше пользователей.

- Отношение количества отправленных пользователю рассылок к числу просмотренных рассылок (SENT_TO_OPENED_RELATION). Меньше соотношение - больше пользователей (диапазон значений 0 - 1 с шагом 0.1)

- Нижний порог уровня активности пользователей (ACTIVITY_LEVEL). Ниже порог - больше пользователей (диапазон значений - от 0.001 с шагом 0.001).
