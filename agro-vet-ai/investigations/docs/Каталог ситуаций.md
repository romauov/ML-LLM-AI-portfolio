# Каталог ситуаций (для поиска аналогичных расследований)

## Назначение каталога

Каталог ситуаций предназначен для накопления и поиска аналогичных случаев расследования ветеринарных проблем. Основные задачи:

- Поиск похожих случаев по диагнозу и условиям
- Извлечение проверенных решений из прошлого опыта
- Обучение системы на подтвержденных расследованиях
- Документирование процесса принятия решений
- Накопление знаний конкретного предприятия

## Структура хранения случая

Каждый случай расследования сохраняется в виде трех связанных компонентов:

### 1. Карточка группы животных
Полные исходные данные о группе животных и условиях содержания

### 2. Протокол чата (диалог расследования)  
Пошаговый диалог между ветеринаром и системой в процессе расследования

### 3. Итоговая запись
Структурированные выводы и результаты расследования для поиска аналогий

Найдено 2 похожих случая в 2023 году. Оба связаны с переохлаждением после отъема.
Рекомендую проверить работу датчиков температуры в секции.

[15:37] Ветеринар: Согласен. Какие меры рекомендуете?

[15:37] Система: 1) Немедленная замена датчика температуры
2) Повышение температуры до 26°C
3) Антибиотикотерапия больных животных
4) Усиленный контроль остальной группы

[15:40] Ветеринар: Меры приняты. Завершаю расследование.

## 3. Итоговая запись (для поиска аналогий)

**Компактная структура для индексации и поиска:**

```yaml
Случай_ID: "2024-08-15-001"
Дата: "2024-08-15"
Группа_ID: "Секция-3-Группа-15"
Ветеринар: "Иванов П.С."

# ИСХОДНАЯ ПРОБЛЕМА
Формулировка_проблемы: "Падеж поросят с диагнозом пневмония, 12 голов за 2 дня"
Диагноз: "Пневмония"
Возрастная_группа: "35-45 дней"
Масштаб: "12 из 85 голов (14.1%)"

# УСТАНОВЛЕННАЯ ПРИЧИНА
Первопричина: "Неисправность датчика температуры"
Механизм: "Неточные показания → переохлаждение → стресс на фоне отъема → пневмония"

# КЛЮЧЕВЫЕ УСЛОВИЯ
Условия:
  - "Недавний отъем (5 дней назад)"
  - "Резкое похолодание наружного воздуха" 
  - "Неисправность датчика температуры"
  - "Молодой возраст (35-45 дней)"
  - "Задержка включения отопления"

# ОПИСАНИЕ РАССЛЕДОВАНИЯ
Комментарий: "Ключевым моментом стало обнаружение временной связи между похолоданием и появлением симптомов. При проверке технических систем выявлена неисправность датчика - он показывал 24°C при фактических 19°C. Это объяснило задержку включения отопления. Аналогичные случаи на предприятии в 2023 году подтвердили гипотезу о связи переохлаждения с пневмонией у недавно отнятых поросят."

# РЕЗУЛЬТАТ РАССЛЕДОВАНИЯ
Метод_установления: "Анализ временной последовательности + проверка оборудования + исторические аналоги"
Достоверность: "Высокая"
Эффективность_решения: "Проблема устранена за 48 часов"
```

## Принципы поиска аналогий

### 1. Критерии поиска по диагнозу
- **Точное совпадение диагноза** - приоритет 1
- **Возрастная группа** - допустимое отклонение ±10 дней
- **Масштаб проблемы** - процент поражения группы

### 2. Критерии поиска по условиям
- **Совпадение ключевых условий** - не менее 50% условий
- **Временная близость событий** - аналогичные предшествующие события
- **Контекст содержания** - тип помещения, плотность посадки

### 3. Алгоритмы ранжирования
```python
Весовые_коэффициенты_поиска = {
    "точное_совпадение_диагноза": 0.4,
    "совпадение_возрастной_группы": 0.2,
    "совпадение_ключевых_условий": 0.3,
    "подтвержденная_достоверность": 0.1
}
```

### 4. Пример результата поиска
```yaml
Запрос: "Пневмония, 30-40 дней, после отъема"

Результаты:
  - Случай_ID: "2023-10-12-003"
    Совпадение: 85%
    Ключевые_сходства: ["диагноз", "возраст", "отъем"]
    Установленная_причина: "Высокая влажность + стресс отъема"
    Комментарий: "Аналогичная ситуация осенью..."
    
  - Случай_ID: "2024-01-15-007"
    Совпадение: 72%
    Ключевые_сходства: ["диагноз", "возраст"]
    Установленная_причина: "Некачественный корм"
    Комментарий: "Схожие симптомы, но другая причина..."
```

## Использование каталога в системе

### При расследовании нового случая:
1. **Поиск аналогов** - система автоматически ищет похожие случаи
2. **Предложение гипотез** - на основе найденных аналогов
3. **Сравнение условий** - выявление сходств и различий
4. **Адаптация решений** - корректировка проверенных методов

### При накоплении опыта:
1. **Автоматическое сохранение** - после завершения расследования
2. **Индексация** - по ключевым параметрам для быстрого поиска
3. **Обновление весов** - корректировка важности факторов
4. **Выявление паттернов** - обнаружение системных проблем

### Метрики качества каталога:
- **Покрытие диагнозов** - процент случаев с найденными аналогами
- **Точность аналогий** - соответствие найденных решений
- **Полнота описаний** - качество сохраненной информации
- **Актуальность** - свежесть данных и их релевантность

## Преимущества каталога:
- Ускорение расследования за счет использования прошлого опыта
- Повышение точности диагностики через сравнение с подтвержденными случаями  
- Накопление знаний конкретного предприятия
- Выявление системных проблем через анализ повторяющихся паттернов
- Документирование процесса принятия решений для обучения персонала
