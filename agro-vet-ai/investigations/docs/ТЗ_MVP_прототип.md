# Техническое задание: MVP прототип системы расследования ветеринарных инцидентов

## 1. Цель MVP

Создать минимальный рабочий прототип системы для расследования ветеринарных инцидентов на базе MCP-сервера и AI-агента с доступом к базе знаний через векторный поиск (RAG).

**Основная ценность MVP:**
- Проверка концепции использования AI + база знаний для помощи в расследовании
- Возможность тестирования с реальными ветеринарами через удобный интерфейс (Claude Desktop/Code)
- Быстрая разработка без необходимости создания веб-интерфейса
- Простота итерации и улучшения промптов агента

## 2. Что НЕ входит в MVP

❌ Сохранение завершенных расследований в БД (каталог ситуаций)
❌ Поиск похожих случаев в каталоге
❌ Веб-интерфейс
❌ REST API
❌ Аутентификация и авторизация
❌ Управление сессиями расследований
❌ Метрики и аналитика

## 3. Компоненты MVP

### 3.1 MCP Server

**Назначение:** Предоставить LLM-агенту доступ к базе знаний через набор инструментов.

**Технологии:**
- Python 3.11+
- MCP SDK (`mcp` package)
- `asyncpg` для работы с PostgreSQL

**Предоставляемые инструменты:**

### 3.2 MCP Инструменты для работы с базой знаний

#### 3.2.1 vet_search

**Функция:** Семантический поиск релевантной информации в базе знаний.

**Входные параметры:**
```json
{
  "query": "string, обязательный - поисковый запрос",
  "top_k": "integer, опциональный, по умолчанию 5 - количество результатов",
  "source_filter": "string, опциональный - фильтр по источнику документа"
}
```

**Выходные данные:**
```json
{
  "results": [
    {
      "content": "текст найденного чанка",
      "source_document": "название книги/источника",
      "page_number": 42,
      "chapter_title": "название главы",
      "similarity_score": 0.87
    }
  ]
}
```

**Алгоритм работы:**
1. Принять поисковый запрос от агента
2. Сгенерировать embedding для запроса (через OpenAI API)
3. Выполнить векторный поиск в таблице `knowledge_base_chunks`
4. Вернуть топ-K наиболее релевантных чанков
5. Отсортировать по similarity score

#### 3.2.3 source_info

**Функция:** Получить подробную информацию об источнике, включая оглавление (список глав).

**Входные параметры:**
```json
{
  "source_document": "string, обязательный - название источника"
}
```

**Выходные данные:**
```json
{
  "source_document": "Antimicrobial Therapy in Veterinary Medicine, 5th Edition",
  "page_range": "1-856",
  "chapters": [
    {
      "chapter_title": "Principles of Antimicrobial Therapy",
      "page_range": "1-45",
    },
    {
      "chapter_title": "Bacterial Infections in Swine",
      "page_range": "156-234",
    }
  ]
}
```

**SQL запрос:**
```sql
-- Получить список глав с диапазонами страниц
SELECT
    chapter_title,
    MIN(page_number) || '-' || MAX(page_number) as page_range,
FROM knowledge_base_chunks
WHERE source_document = :source_document
    AND chapter_title IS NOT NULL
GROUP BY chapter_title
ORDER BY MIN(page_number);
```

#### 3.2.4 get_pages

**Функция:** Получить контент с конкретных страниц или диапазона страниц из источника.

**Входные параметры:**
```json
{
  "source_document": "string, обязательный - название источника",
  "page_start": "integer, обязательный - начальная страница",
  "page_end": "integer, опциональный - конечная страница (если не указана, то только page_start)"
}
```

**Выходные данные:**
```json
{
  "source_document": "Antimicrobial Therapy in Veterinary Medicine, 5th Edition",
  "page_range": "42-45",
  "pages": [
    {
      "page_number": 42,
      "chapter_title": "Bacterial Infections in Swine",
      "content": "текст страницы 42",
    },
    {
      "page_number": 43,
      "chapter_title": "Bacterial Infections in Swine",
      "content": "текст страницы 43",
    }
  ]
}
```

**Примеры использования:**

```python
# Получить только страницу 42
get_pages(source_document="Antimicrobial Therapy...", page_start=42)

# Получить страницы 42-45
get_pages(source_document="Antimicrobial Therapy...", page_start=42, page_end=45)
```

## 4. Функциональные требования MVP

### 4.1 Работа с базой знаний

#### 4.1.1 Семантический поиск

**FR-1.1:** Агент должен иметь возможность искать информацию по произвольному текстовому запросу
**FR-1.2:** Поиск должен быть семантическим (векторное сходство), а не текстовым
**FR-1.3:** Результаты должны содержать исходный контент и метаданные (источник, страница, глава)
**FR-1.4:** Количество возвращаемых результатов должно быть настраиваемым (top_k)
**FR-1.5:** Должна быть возможность фильтрации по источнику документа

#### 4.1.2 Навигация по источникам

**FR-1.6:** Агент должен иметь возможность получить список всех доступных источников
**FR-1.7:** Агент должен иметь возможность просмотреть оглавление (список глав) любого источника
**FR-1.8:** Агент должен иметь возможность получить контент с конкретных страниц или диапазона страниц
**FR-1.9:** При получении страниц должна сохраняться последовательность чанков (по chunk_number)

#### 4.1.3 Сценарии использования инструментов

**Сценарий 1: Обзорное знакомство с базой знаний**
1. `vet_sources()` - увидеть, какие источники доступны с кратким описанием
2. `source_info("Antimicrobial Therapy...")` - посмотреть оглавление интересующей книги
3. `vet_search("E.coli treatment")` - найти релевантные разделы

**Сценарий 2: Углубленное изучение темы**
1. `vet_search("neonatal diarrhea", top_k=3)` - найти релевантные места
2. Результат: найдено на стр. 156-158 в главе "Bacterial Infections in Swine"
3. `get_pages(source_document="...", page_start=156, page_end=160)` - прочитать полный контекст

**Сценарий 3: Работа с конкретным источником**
1. `source_info("Болезни свиней")` - посмотреть оглавление
2. Видим главу "Диарея поросят" на страницах 42-67
3. `get_pages(source_document="Болезни свиней", page_start=42, page_end=67)` - прочитать всю главу

### 4.2 Процесс расследования

**FR-2.1:** Агент должен принимать на вход:
- Описание инцидента (текст)
- Карточку группы животных (Markdown, может быть в виде файла или текста)

**FR-2.2:** Агент должен создавать структуру расследования:
- Создать папку в `investigations/YYYY-MM-DD_short-name/`
- Сохранить входные данные в файлы
- Вести рабочие заметки в процессе анализа
- Сформировать итоговый отчет

**FR-2.3:** Агент должен следовать структурированному процессу:
1. Анализ входных данных
2. Определение типа проблемы (диарея, респираторная, и т.д.)
3. Открытие соответствующей инструкции из `instructions/` при необходимости
4. Поиск релевантной информации в базе знаний через MCP tool
5. Генерация гипотез с обоснованием
6. Ранжирование гипотез по вероятности
7. Формулирование уточняющих вопросов
8. Итеративное уточнение гипотез на основе ответов

**FR-2.4:** Агент должен использовать информацию из базы знаний для:
- Идентификации возможных патогенов
- Рекомендаций по диагностике
- Предложений по терапии
- Обоснования гипотез
- Сохранение найденных релевантных цитат в `knowledge_snippets.md`

**FR-2.5:** Агент должен генерировать гипотезы в структурированном формате:
```
**Гипотеза N**: [Название]
**Уверенность**: high/medium/low
**Аргументация**: [Конкретные факты из карточки + ссылки на знания из базы]
**Источники из базы знаний**: [Ссылки на документы и страницы]
**Рекомендации**: [Что проверить/как подтвердить]
```

### 4.3 Работа с контекстом

**FR-3.1:** Агент должен опираться только на:
- Факты из описания инцидента
- Данные из карточки группы
- Информацию из базы знаний

**FR-3.2:** Агент НЕ должен:
- Выдумывать факты
- Делать предположения о данных, которых нет
- Игнорировать информацию из базы знаний

**FR-3.3:** Если информации недостаточно, агент должен:
- Явно это признать
- Задать конкретные уточняющие вопросы
- Объяснить, зачем нужна дополнительная информация

## 5. Технические требования

### 5.1 MCP Server

**TR-1.1:** Сервер должен быть реализован на Python 3.11+
**TR-1.2:** Сервер должен использовать официальный MCP SDK
**TR-1.3:** Сервер должен работать как stdio-сервер (для подключения из Claude Desktop/Code)
**TR-1.4:** Подключение к PostgreSQL должно быть асинхронным (asyncpg)
**TR-1.5:** Параметры подключения к БД должны быть вынесены в конфигурацию (переменные окружения)

### 5.2 Генерация Embeddings

**TR-2.1:** Использовать модель OpenAI `text-embedding-3-large` (размерность 1536)
**TR-2.2:** API ключ OpenAI должен быть в переменных окружения
**TR-2.3:** Обработка ошибок API (rate limits, timeout)

### 5.3 Векторный поиск

**TR-3.1:** Использовать pgvector для поиска по косинусному сходству
**TR-3.2:** Поисковый запрос:
```sql
SELECT
    content,
    content_type,
    content_name,
    source_document,
    page_number,
    chapter_title,
    keywords,
    1 - (embedding <=> query_embedding) as similarity_score
FROM knowledge_base_chunks
ORDER BY embedding <=> query_embedding
LIMIT top_k
```

**TR-3.3:** Поддержка фильтрации по `source_document` через WHERE clause
**TR-3.4:** Минимальный порог similarity_score: 0.5 (результаты ниже игнорируются)

### 5.4 Обработка ошибок

**TR-4.1:** Graceful handling ошибок подключения к БД
**TR-4.2:** Логирование ошибок и запросов
**TR-4.3:** Информативные сообщения об ошибках для пользователя

## 9. Пример использования

### Входные данные:

**Описание инцидента:**
```
У нас на свинокомплексе в группе поросят-сосунов наблюдается:
- Возраст: 3-7 дней
- Падеж: 8 поросят за 2 дня
- Симптомы: водянистая диарея, обезвоживание
- Карточка группы прилагается
```

**Карточка группы:** (в виде Markdown файла или текста)

### Ожидаемое поведение агента:

1. **Анализ данных:**
   - Возраст: 3-7 дней (неонатальная диарея)
   - Масштаб: 8 падежей за 2 дня (серьезная ситуация)
   - Симптомы: водянистая диарея + обезвоживание

2. **Поиск в базе знаний:**
   - Запрос: "неонатальная диарея поросята E.coli ротавирус"
   - Запрос: "диарея поросята 3-7 дней патогены"
   - Запрос: "водянистая диарея обезвоживание лечение"

3. **Генерация гипотез:**

   **Гипотеза 1**: Энтеротоксигенная E.coli (ETEC)
   **Уверенность**: high
   **Аргументация**: Возраст 3-7 дней и водянистая диарея классически указывают на E.coli. Согласно данным из базы знаний [источник], ETEC является наиболее частой причиной НД в этом возрасте. Быстрое развитие обезвоживания соответствует клинической картине.
   **Рекомендации**: Бактериологический посев кала, антибиотикограмма, проверка вакцинации свиноматок против E.coli.

   **Гипотеза 2**: Ротавирусная инфекция
   **Уверенность**: medium
   **Аргументация**: Ротавирус также часто встречается в возрасте 1-7 дней. База знаний указывает на водянистую диарею как типичный симптом. Часто сочетается с E.coli.
   **Рекомендации**: ПЦР на ротавирус, проверка вакцинации свиноматок.

   **Гипотеза 3**: Технологические нарушения (прием опороса)
   **Уверенность**: medium
   **Аргументация**: Недостаточное получение молозива или переохлаждение могут снизить иммунитет и спровоцировать инфекцию.
   **Рекомендации**: Проверить протокол приема опороса, температуру в секции, работу ковриков обогрева.

4. **Уточняющие вопросы:**
   - Получили ли поросята молозиво в первые 3 часа жизни?
   - Какова температура в станке для поросят? Работают ли коврики обогрева?
   - Вакцинированы ли свиноматки против E.coli и ротавируса?
   - Есть ли признаки мастита у свиноматки?
   - Проводилось ли вскрытие? Какие находки?

## 12. Риски и ограничения MVP

**Риски:**
- Качество ответов зависит от качества промпта агента (требуется итерация)
- Релевантность результатов поиска зависит от качества embeddings и чанкирования
- OpenAI API может быть недоступен или медленный (rate limits)

**Ограничения:**
- Нет персистентности расследований (только в рамках одной сессии Claude)
- Нет истории расследований и поиска аналогов
- Нет многопользовательской работы
- Требуется ручное копирование карточек групп в чат

**Митигация:**
- Тщательная работа над промптом агента с примерами
- Тестирование на реальных кейсах и итерация
- Fallback на локальные embeddings модели при необходимости
- Четкая документация для пользователей
