# Анализ инструментов для ветеринарного агента-расследователя

Этот документ анализирует инструменты из Qwen Code (см. `qwen-code-tools.json`) и определяет, какие из них применимы для нашего ветеринарного агента.

## Источник

**Оригинальные инструменты:** Из [QwenLM/qwen-code](https://github.com/QwenLM/qwen-code)

## Категоризация инструментов

### ✅ Обязательные для ветеринарного агента

Эти инструменты критически важны для работы с файловой структурой расследований:

#### 1. **`read_file`** ✅ НУЖЕН
**Для чего:** Чтение файлов расследования (incident.md, group_card.md, hypotheses.md, etc.)

**Адаптация:** Минимальная
- Используется как есть
- Агент будет читать markdown файлы расследований

**Примеры использования:**
```json
{
  "name": "read_file",
  "parameters": {
    "absolute_path": "/agent-workspace/investigations/2025-01-15_piglet-diarrhea/incident.md"
  }
}
```

---

#### 2. **`write_file`** ✅ НУЖЕН
**Для чего:** Создание новых файлов расследования

**Адаптация:** Минимальная
- Используется для создания файлов типа hypotheses.md, analysis.md, report.md

**Примеры использования:**
```json
{
  "name": "write_file",
  "parameters": {
    "file_path": "/agent-workspace/investigations/2025-01-15_piglet-diarrhea/hypotheses.md",
    "content": "# Гипотезы\n\n## 1. E.coli инфекция (вероятность: высокая)\n..."
  }
}
```

---

#### 3. **`edit`** ✅ НУЖЕН (с адаптацией)
**Для чего:** Обновление существующих файлов расследования

**Адаптация:** Важная особенность
- В контексте расследований часто нужно обновлять секции markdown
- Может потребоваться дополнительный tool `update_file_section` для более удобного обновления markdown секций

**Примеры использования:**
```json
{
  "name": "edit",
  "parameters": {
    "file_path": "/agent-workspace/investigations/2025-01-15_piglet-diarrhea/hypotheses.md",
    "old_string": "## 1. E.coli инфекция (вероятность: средняя)",
    "new_string": "## 1. E.coli инфекция (вероятность: высокая)"
  }
}
```

**Альтернатива:** Создать специализированный `update_markdown_section`:
```json
{
  "name": "update_markdown_section",
  "parameters": {
    "file_path": "/agent-workspace/investigations/2025-01-15_piglet-diarrhea/hypotheses.md",
    "section_heading": "## 1. E.coli инфекция",
    "new_content": "## 1. E.coli инфекция (вероятность: высокая)\n\n**Обоснование:**\n- Возраст группы (3-5 дней) совпадает...\n"
  }
}
```

---

#### 4. **`list_directory`** ✅ НУЖЕН
**Для чего:** Просмотр файлов текущего расследования

**Адаптация:** Используется как есть

**Примеры использования:**
```json
{
  "name": "list_directory",
  "parameters": {
    "path": "/agent-workspace/investigations/2025-01-15_piglet-diarrhea"
  }
}
```

---

#### 5. **`todo_write`** ✅ НУЖЕН
**Для чего:** Планирование шагов расследования и отслеживание прогресса

**Адаптация:** Используется практически как есть, но примеры в description адаптируются под ветеринарный контекст

**Примеры задач для расследования:**
- "Проанализировать описание инцидента"
- "Найти информацию о диарее поросят в базе знаний"
- "Сформировать первичные гипотезы"
- "Обновить файл hypotheses.md"
- "Задать уточняющие вопросы пользователю"

**Адаптированное описание:**
```
Используйте этот инструмент для планирования и отслеживания шагов расследования.

Примеры шагов расследования:
1. Анализ инцидента и карточки группы
2. Поиск информации в базе знаний
3. Формирование гипотез
4. Обновление файлов расследования
5. Формулирование вопросов для уточнения
```

---

### 🟡 Полезные, но не критичные

#### 6. **`glob`** 🟡 МОЖЕТ ПРИГОДИТЬСЯ
**Для чего:** Поиск файлов по паттерну (например, найти все расследования за определенный период)

**Адаптация:** Может быть полезен для:
- Поиска похожих расследований: `glob("**/incident.md")`
- Поиска расследований по типу проблемы

**Решение:** Включить, но с низким приоритетом

---

#### 7. **`search_file_content`** 🟡 МОЖЕТ ПРИГОДИТЬСЯ
**Для чего:** Поиск по содержимому файлов расследований

**Адаптация:** Может использоваться для:
- Поиска ключевых слов в прошлых расследованиях
- Поиска упоминаний конкретных патогенов/препаратов

**Решение:** Включить, но с низким приоритетом

---

#### 8. **`read_many_files`** 🟡 МОЖЕТ ПРИГОДИТЬСЯ
**Для чего:** Чтение нескольких файлов расследования одновременно

**Адаптация:** Может быть полезен для:
- Чтения всех файлов расследования для формирования отчета
- Анализа нескольких похожих расследований

**Решение:** Включить, но можно реализовать через несколько вызовов `read_file`

---

### ❌ Не нужны для ветеринарного агента

#### 9. **`run_shell_command`** ❌ НЕ НУЖЕН
**Почему:** Ветеринарный агент работает только с файлами и базой знаний, не требует выполнения shell команд

---

#### 10. **`task`** ❌ НЕ НУЖЕН (изначально)
**Почему:** Субагенты не критичны для MVP. Весь функционал можно реализовать в основном агенте.

**Возможное будущее использование:**
- Специализированный агент для анализа кормов
- Специализированный агент для респираторных проблем

**Решение:** Не включать в MVP, рассмотреть для расширенной версии

---

#### 11. **`save_memory`** ❌ НЕ НУЖЕН
**Почему:** Информация о ферме/группе сохраняется в файлах расследования, не требуется отдельный механизм памяти

**Альтернатива:** Вся необходимая информация хранится в:
- `group_card.md` - информация о группе животных
- `incident.md` - описание проблемы
- `analysis.md` - рабочие заметки агента

---

#### 12. **`exit_plan_mode`** ❌ НЕ НУЖЕН
**Почему:** Режим планирования не применим к расследованиям. Агент сразу начинает работать с инцидентом.

---

#### 13. **`web_fetch`** ❌ НЕ НУЖЕН
**Почему:** Вся информация берется из локальной базы знаний через MCP-инструменты `vet_search`, `get_pages`.

---

## Итоговый набор инструментов для ветеринарного агента

### Backend-инструменты (через FastAPI)

Эти инструменты реализуются на стороне backend и предоставляются агенту как OpenAI function calls:

1. **`read_file`** - чтение файла
2. **`write_file`** - создание файла
3. **`edit`** - редактирование файла (или `update_markdown_section`)
4. **`list_directory`** - список файлов в папке
5. **`todo_write`** - управление задачами

### MCP-инструменты (через mcp-server и mcp-filesystem)

Эти инструменты предоставляются MCP-серверами:

#### Из `mcp-server` (vetretro) - база знаний:
6. **`vet_search`** - семантический поиск
7. **`vet_sources`** - список источников
8. **`source_info`** - информация об источнике
9. **`get_pages`** - извлечение страниц

#### Из `mcp-filesystem` - файловая система расследований:
10. **`create_investigation`** - создание структуры расследования
11. **`list_investigations`** - список всех расследований
12. **`list_files`** - список файлов расследования (дублирует list_directory)

### Специальные инструменты

13. **`respond_in_schema`** - для механизма auto-continue (см. `auto-continue-mechanism-example.md`)

## Приоритизация реализации

### Этап 1: MVP (минимальный набор)
1. `read_file`
2. `write_file`
3. `list_directory`
4. `todo_write`
5. `vet_search` (MCP)
6. `get_pages` (MCP)

### Этап 2: Базовая версия
7. `edit` или `update_markdown_section`
8. `create_investigation` (MCP)
9. `vet_sources` (MCP)
10. `source_info` (MCP)

### Этап 3: Расширенная версия
11. `respond_in_schema` (auto-continue)
12. `glob` (опционально)
13. `search_file_content` (опционально)
14. `read_many_files` (опционально)

## Технические детали реализации

### Схема взаимодействия

```
User
  ↓
Web Frontend (React/Open WebUI)
  ↓
FastAPI Backend (/v1/chat/completions)
  ↓
OpenRouter LLM (Qwen)
  ↓ (function calls)
  ├─→ Backend Tools (read_file, write_file, edit, list_directory, todo_write)
  ├─→ MCP Server: vetretro via SSE (vet_search, get_pages, vet_sources, source_info)
  └─→ MCP Server: mcp-filesystem via SSE (create_investigation, list_investigations, list_files)
```

### Формат function calling

Backend преобразует MCP-инструменты в OpenAI function format:

```python
# MCP tool от vetretro сервера
{
  "type": "function",
  "function": {
    "name": "vet_search",
    "description": "Semantic search in veterinary knowledge base",
    "parameters": {
      "type": "object",
      "properties": {
        "query": {"type": "string", "description": "Search query"},
        "limit": {"type": "integer", "default": 5}
      },
      "required": ["query"]
    }
  }
}

# Backend tool
{
  "type": "function",
  "function": {
    "name": "read_file",
    "description": "Read investigation file",
    "parameters": {
      "type": "object",
      "properties": {
        "absolute_path": {"type": "string"}
      },
      "required": ["absolute_path"]
    }
  }
}
```

## Отличия от Qwen Code

| Аспект | Qwen Code | Ветеринарный агент |
|--------|-----------|-------------------|
| **Домен** | Разработка ПО | Ветеринарные расследования |
| **Файлы** | Код (JS/TS/Python/etc) | Markdown файлы расследований |
| **Внешние источники** | Web, документация | База знаний (RAG) |
| **Shell команды** | Используются (npm, git, etc) | Не используются |
| **Субагенты** | Есть (code-reviewer, etc) | Нет в MVP |
| **Память** | save_memory для preferences | Информация в файлах расследования |
| **Верификация** | Тесты, сборка, линтеры | Проверка полноты информации, цитирование источников |
