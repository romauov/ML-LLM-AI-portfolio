# VetRetro Agent Workspace

Рабочее пространство для AI-ассистента ветеринарного врача по расследованию инцидентов в промышленном свиноводстве.

## Описание

Agent Workspace - это структурированная среда для проведения ветеринарных расследований с помощью AI-ассистента, имеющего доступ к базе знаний ветеринарной литературы.

**Ключевые возможности:**
- Систематический подход к расследованию инцидентов
- Доступ к научной ветеринарной литературе через MCP-сервер
- Структурированное документирование каждого этапа
- Формирование обоснованных гипотез с опорой на научные источники
- Конкретные, действенные рекомендации

## Как использовать

### 1. Создание новой папки расследования

Для каждого нового расследования создайте отдельную папку:

```bash
mkdir investigations/2025-01-15_farm-ivanovka_diarrhea
cd investigations/2025-01-15_farm-ivanovka_diarrhea
```

**Рекомендуемый формат имени папки:**
`YYYY-MM-DD_farm-name_problem-type`

### 2. Запуск AI-ассистента

Запустите Claude Code из папки расследования:

```bash
claude
# или
claude-code
```

**Важно:** Файл `AGENTS.md` из корня agent-workspace автоматически загрузится как системный промпт для агента.

### 3. Начало расследования

Предоставьте агенту описание инцидента. Агент автоматически:
1. Создаст структуру файлов для расследования
2. Запросит недостающую информацию
3. Поможет заполнить карточку группы (используя шаблон из `templates/`)
4. Определит тип проблемы
5. Загрузит соответствующую инструкцию из `instructions/`
6. Проведет систематическое расследование

### 4. Работа с чек-листами

Агент будет создавать чек-листы для сбора данных. Если требуется выезд на ферму, агент создаст файл `03a_field_checklist.md` - готовый к печати чек-лист.

### 5. Асинхронная работа

Расследование может занимать дни или недели (в ожидании лабораторных данных). Агент:
- Сформулирует предварительные гипотезы на основе доступных данных
- Обновит файл `STATUS.md` с текущим прогрессом
- После получения новых данных - продолжит расследование

Просто возвращайтесь к агенту с новыми данными, и он продолжит с того места, где остановился.

```

├── README.md                  # Этот файл
├── instructions/              # Специализированные руководства
│   ├── neonatal_diarrhea.md   # Расследование неонатальной диареи
│   ├── respiratory.md         # Респираторные проблемы
│   └── prrs.md                # РРСС

│   ├── group_card_template.md # Шаблон карточки группы
```

## Структура файлов расследования

При работе с агентом в папке расследования будут создаваться следующие файлы:

```
investigation_folder/
├── STATUS.md                   # Текущий статус и прогресс
├── 00_incident.md              # Описание инцидента
├── 01_group_card.md            # Карточка группы
├── 02_checklist_general.md     # Общий чек-лист
├── 03_checklist_specific.md    # Специализированный чек-лист
├── 03a_field_checklist.md      # Чек-лист для выезда (если нужен)
├── 04_clinical_data.md         # Клинические данные
├── 05_lab_results.md           # Лабораторные результаты
├── 06_knowledge_snippets.md    # Цитаты из базы знаний
├── 07_hypotheses.md            # Гипотезы с обоснованием
├── 08_conclusions.md           # Выводы и рекомендации
└── 09_report.md                # Финальный отчет
```

## Типы поддерживаемых расследований

### Неонатальная диарея
- **Возраст:** 1-15 дней
- **Инструкция:** `instructions/neonatal_diarrhea.md`
- **Ключевые этапы:** Технологические факторы → Инфекционные → Терапия/профилактика → Синтез

### Респираторные проблемы
- **Возраст:** Любой (чаще доращивание/откорм)
- **Инструкция:** `instructions/respiratory.md`
- **Ключевые этапы:** Производственные показатели → Полевая проверка → Клиника и лаборатория → Терапия → Первопричины → План действий

### РРСС (Репродуктивно-респираторный синдром)
- **Признаки:** Аборты + респираторная клиника
- **Инструкция:** `instructions/prrs.md`
- **Ключевые этапы:** Сбор данных → Клиника и вскрытие → Диагностический мониторинг → Определение статуса стада → План мероприятий

## Доступ к базе знаний

Агент имеет доступ к ветеринарной базе знаний через MCP-инструменты:

- **vet_search** - семантический поиск по базе знаний
- **vet_sources** - список доступных источников
- **source_info** - оглавление конкретного источника
- **get_pages** - чтение страниц из источников

Агент **проактивно** использует базу знаний на каждом этапе расследования для:
- Поиска информации о патогенах
- Рекомендаций по диагностике и лечению
- Обоснования гипотез научными данными
- Проверки дифференциальных диагнозов

Все находки из базы знаний сохраняются в файл `06_knowledge_snippets.md` с указанием источников и страниц.

## Примеры использования

### Пример 1: Начало нового расследования

```bash
# Создать папку расследования
mkdir investigations/2025-01-20_farm-severny_respiratory
cd investigations/2025-01-20_farm-severny_respiratory

# Запустить агента
claude

# В чате:
# "У нас на ферме Северный в секции доращивания массовый кашель
# у поросят 40-60 дней. Падеж 5 голов за неделю."
```

Агент:
1. Создаст файл `00_incident.md` с описанием
2. Запросит карточку группы или поможет заполнить
3. Определит тип: Респираторная проблема
4. Загрузит `instructions/respiratory.md`
5. Начнет систематическое расследование

### Пример 2: Продолжение расследования после получения лаб. данных

```bash
# Вернуться в папку расследования
cd investigations/2025-01-20_farm-severny_respiratory

# Запустить агента
claude

# В чате:
# "Получил результаты лаборатории: выделен Mycoplasma hyopneumoniae,
# Streptococcus suis. Антибиотикограмма во вложении."
```

Агент:
1. Прочитает `STATUS.md` для понимания текущего состояния
2. Добавит результаты в `05_lab_results.md`
3. Обновит гипотезы в `07_hypotheses.md`
4. Проведет поиск в базе знаний по выявленным патогенам
5. Сформулирует рекомендации

## Советы по эффективной работе

1. **Будьте конкретны:** Предоставляйте точные данные (возраст, количество, даты)
2. **Используйте карточки групп:** Заполняйте шаблон `templates/group_card_template.md`
3. **Сохраняйте документы:** Прикрепляйте файлы результатов анализов
4. **Следуйте чек-листам:** Агент создает чек-листы для систематической работы
5. **Обновляйте STATUS.md:** Агент автоматически обновляет прогресс
6. **Задавайте вопросы:** Агент - ваш коллега, спрашивайте что непонятно

## Ограничения

- Агент не заменяет ветеринарного врача, а ассистирует ему
- Все рекомендации требуют профессиональной оценки
- База знаний ограничена загруженными источниками
- Для критических решений консультируйтесь с коллегами
