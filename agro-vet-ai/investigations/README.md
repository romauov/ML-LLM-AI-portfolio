# VetRetro - AI-система для расследования ветеринарных инцидентов

> **VetRetro** - это интеллектуальная система для систематического расследования инцидентов (вспышки болезней, падеж, снижение продуктивности) в промышленном свиноводстве и птицеводстве, основанная на RAG (Retrieval-Augmented Generation) и специализированной ветеринарной базе знаний.

## 🎯 Что это такое?

**VetRetro** помогает ветеринарным специалистам проводить **систематические расследования** проблем на животноводческих комплексах:

### Проблема
Ветеринар сталкивается с инцидентом (диарея у поросят, падеж, респираторные проблемы), и нужно:
- Быстро найти похожие случаи в литературе
- Сформировать ранжированные гипотезы
- Назначить правильные анализы
- Дать обоснованные рекомендации
- Задокументировать расследование

### Решение
VetRetro предоставляет:
- **AI-ассистента**, который работает как коллега-эксперт
- **Доступ к ветеринарной базе знаний** через семантический поиск
- **Структурированный процесс расследования** с чек-листами
- **Автоматическую документацию** всех этапов
- **Асинхронную работу** (расследование может длиться дни/недели)

## ✨ Возможности

### Для ветеринара:

✅ **Систематический подход**
- Структурированные чек-листы для сбора данных
- Этапы расследования от простого к сложному
- Ничего не упускается

✅ **Доступ к знаниям**
- Семантический поиск по 4 ветеринарным книгам (3,451 фрагментов)
- Цитаты с номерами страниц
- Информация о лечении, диагностике, профилактике

✅ **Обоснованные выводы**
- Гипотезы с уровнями уверенности (high/medium/low)
- Рекомендации подкреплены научными источниками
- Учёт многофакторности проблем

✅ **Полная документация**
- Все данные сохраняются в markdown файлах
- Можно возобновить расследование в любой момент
- Финальный отчёт генерируется автоматически

### Типы расследований:

- **Неонатальная диарея** (поросята 0-7 дней)
- **Постотъёмная диарея** (поросята 21-42 дня)
- **Респираторные проблемы** (кашель, одышка, любой возраст)
- **PRRS** (репродуктивно-респираторный синдром)
- Другие инциденты (универсальный протокол)

## 🏗️ Архитектура

VetRetro состоит из двух основных компонентов:

```
VetRetro/
├── mcp-server/              # MCP сервер для доступа к базе знаний
│   ├── src/
│   │   ├── server.py        # Главный MCP сервер
│   │   ├── knowledge_base.py # Логика RAG поиска
│   │   ├── embeddings.py    # Генерация эмбеддингов
│   │   └── config.py        # Конфигурация
│   └── README.md
│
└── agent-workspace/         # Рабочая среда для AI-агента
    ├── AGENTS.md            # Главный промпт агента (автозагрузка)
    ├── instructions/        # Специализированные инструкции
    │   ├── neonatal_diarrhea.md
    │   ├── respiratory.md
    │   └── prrs.md
    ├── templates/           # Шаблоны для расследований
    ├── examples/            # Примеры завершённых расследований
    └── investigations/      # Активные расследования
```

### 1. MCP Сервер (Model Context Protocol)

**Роль:** Предоставляет AI-агенту доступ к ветеринарной базе знаний.

**Инструменты:**
- `vet_search` - Семантический поиск по базе знаний
- `vet_sources` - Список доступных источников
- `source_info` - Оглавление книги
- `get_pages` - Получение контента со страниц

**Технологии:**
- Python 3.11+
- PostgreSQL + pgvector (векторный поиск)
- OpenAI Embeddings API (через VseGPT)
- MCP SDK

### 2. Agent Workspace

**Роль:** Структурированная среда для проведения расследований.

**Ключевые файлы:**
- `AGENTS.md` - Промпт агента (автоматически загружается при запуске)
- `instructions/*.md` - Специализированные инструкции по типам проблем
- `templates/*.md` - Шаблоны для документов

**Процесс расследования:**
1. Агент создаёт папку `investigations/YYYY-MM-DD_название/`
2. Собирает данные по чек-листам
3. Ищет информацию в базе знаний (проактивно)
4. Формирует гипотезы с обоснованиями
5. Обновляет файлы на каждом этапе
6. Генерирует финальный отчёт

## 🚀 Быстрый старт

### Требования

- Python 3.11+
- PostgreSQL 14+ с расширением pgvector
- API ключ для эмбеддингов (VseGPT)
- Claude Code или другой AI-ассистент с поддержкой MCP

### 1. Установка MCP сервера

```bash
cd mcp-server
pip install -e .
```

### 2. Настройка окружения

Создайте файл `.env` в корне проекта:

```env
# База данных
DB_HOST=10.0.3.123
DB_PORT=5432
DB_NAME=vetbot
DB_USER=vetbot
DB_PASSWORD=vetbot

# API для эмбеддингов
VSEGPT_API_KEY=sk-or-vv-your-key-here
OPENAI_API_BASE=https://api.vsegpt.ru/v1
```

### 3. Подключение к базе данных

```bash
PGPASSWORD=vetbot psql -h 10.0.3.123 -U vetbot -d vetbot
```

### 4. Запуск MCP сервера

**Вариант A: HTTP/SSE транспорт**
```bash
cd mcp-server
python -m src.server
# Сервер запустится на http://localhost:3000
```

**Вариант B: stdio транспорт (для Claude Desktop)**
```bash
python -m src.server --transport stdio
```

### 5. Настройка Claude Code

Добавьте в конфигурацию MCP серверов:

```json
{
  "mcpServers": {
    "vetretro": {
      "command": "python",
      "args": ["-m", "src.server", "--transport", "stdio"],
      "cwd": "/path/to/VetRetro/mcp-server",
      "env": {
        "VSEGPT_API_KEY": "your-key-here"
      }
    }
  }
}
```

### 6. Начало работы

```bash
cd agent-workspace
# Запустите Claude Code из этой директории
# AGENTS.md загрузится автоматически
```

Отправьте агенту описание инцидента:
```
У меня на ферме "Рассвет" проблема. Поросята-сосуны 3-7 дней,
водянистая диарея. За 2 дня пало 8 голов из 45. Помоги разобраться.
```

Агент начнёт систематическое расследование! 🎉

Агент использует vet_search для поиска информации о:
- E.coli постотъёмная диарея
- Стресс отъёма
- Антибиотикотерапия
```

**Шаг 5: Вскрытие и лабораторные данные**
```
Пользователь: При вскрытии желудок пустой, катаральный энтерит...

### База данных (PostgreSQL + pgvector)

**Подключение:**
```bash
PGPASSWORD=vetbot psql -h 10.0.3.123 -U vetbot -d vetbot
```

**Таблица knowledge_base_chunks:**
- `content` (text) - Текстовый фрагмент
- `embedding` (vector(1536)) - Векторное представление
- `source_document` (text) - Название книги
- `page_number` (int) - Номер страницы
- `chapter_title` (text) - Название главы
- `keywords` (text[]) - Ключевые слова

**Поиск по сходству:**
```sql
SELECT content, source_document, page_number,
       1 - (embedding <=> query_embedding) as similarity
FROM knowledge_base_chunks
ORDER BY embedding <=> query_embedding
LIMIT 10;
```

### API для эмбеддингов (VseGPT)

```python
from openai import OpenAI

client = OpenAI(
    api_key=os.getenv("VSEGPT_API_KEY"),
    base_url="https://api.vsegpt.ru/v1",
)

response = client.embeddings.create(
    model="text-embedding-ada-002",
    input="E.coli диарея лечение",
    encoding_format="float"
)

embedding = response.data[0].embedding  # 1536-мерный вектор
```
