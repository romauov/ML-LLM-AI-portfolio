# Итоговая реализация функции extract_document

## Что реализовано

### 1. MCP инструмент `extract_document`

**Параметры:**
- `file_path` - путь к файлу (PDF или DOCX)
- ~~`model`~~ - убран для экономии контекста

**Автоматическое поведение:**
- PDF → `utils/pdf-ocr-1.0` (с извлечением изображений)
- DOCX → `utils/extract-text-1.0`

### 2. Структура сохранения

```
extracted_documents/
└── <имя_документа>/
    ├── page_001.md
    ├── page_002.md
    ├── ...
    ├── img-0.jpeg
    └── img-1.jpeg
```

**Для PDF (OCR модель):**
- Каждая страница → отдельный MD файл
- Изображения извлекаются в base64 и сохраняются как JPEG
- Markdown таблицы, структура сохраняется

**Для DOCX:**
- Один файл `content.md` со всем текстом

### 3. Тестирование

**Тестовые файлы:**
1. `IFA.pdf` (91KB) - лабораторные результаты ИФА
   - ✅ 1 страница извлечена
   - ✅ 1 изображение (график) сохранено (69KB)
   - ✅ Таблица с данными в markdown

2. `test_document.docx` (36KB) - договор
   - ✅ Полный текст извлечен
   - ✅ Структура документа сохранена

**Время обработки:**
- PDF (с OCR): ~3-5 секунд
- DOCX: ~2-3 секунды

### 4. Метаданные возврата

```json
{
  "success": true,
  "text": "полный текст",
  "metadata": {
    "filename": "IFA.pdf",
    "file_size_bytes": 94029,
    "model_used": "utils/pdf-ocr-1.0",
    "output_dir": "extracted_documents/ifa",
    "saved_pages": ["ifa/page_001.md"],
    "saved_images": ["ifa/img-0.jpeg"],
    "pages_processed": 1,
    "has_images": true,
    "images_count": 1
  }
}
```

### 5. Обработка ошибок

- ✅ Неподдерживаемый формат (XLSX → 500 error)
- ✅ Файл не найден
- ✅ Rate limiting (автоматический retry с задержкой)
- ✅ Кириллица в именах файлов (транслитерация)

### 6. Интеграция с agent workflow

**Сценарий использования:**
1. Ветеринар загружает PDF с результатами ПЦР
2. Агент вызывает `extract_document("/path/to/results.pdf")`
3. MCP сервер:
   - Извлекает текст через VseGPT API
   - Сохраняет страницы в `extracted_documents/results/`
   - Извлекает изображения (таблицы, графики)
4. Агент получает:
   - Краткое содержимое (первые 1000 символов)
   - Пути к сохраненным файлам
5. Агент может:
   - Прочитать полные страницы через Read tool
   - Сохранить данные в `05_lab_results.md`
   - Обновить гипотезы на основе результатов

## Особенности реализации

### Автоматический выбор модели
```python
if ext == ".pdf":
    return "utils/pdf-ocr-1.0"  # OCR для лучшего качества таблиц
elif ext == ".docx":
    return "utils/extract-text-1.0"
```

### Извлечение изображений
```python
if selected_model == "utils/pdf-ocr-1.0":
    payload["return_images"] = True  # Всегда для PDF
```

### Обработка страниц
- OCR модель возвращает `pages` массив с markdown для каждой страницы
- Каждая страница сохраняется отдельно с префиксом `page_XXX.md`
- Изображения декодируются из base64 и сохраняются

### Rate limiting
- Автоматический retry с экспоненциальной задержкой (1s, 2s, 4s)
- Максимум 3 попытки
- Логирование всех retry

## Примеры использования

### Пример 1: Лабораторные результаты ПЦР

```python
result = await extractor.extract_text("/home/vet/pcr_results.pdf")

# Результат:
# extracted_documents/pcr_results/
#   ├── page_001.md (результаты анализа)
#   ├── page_002.md (расшифровка)
#   └── img-0.jpeg (таблица с данными)
```

### Пример 2: Договор с лабораторией

```python
result = await extractor.extract_text("/home/vet/contract.docx")

# Результат:
# extracted_documents/contract/
#   └── content.md (весь договор)
```

## Преимущества подхода

1. **Экономия контекста** - возвращаем только краткое содержимое (1000 символов)
2. **Полный доступ** - агент может прочитать любую страницу через Read tool
3. **Изображения сохранены** - графики и таблицы доступны для анализа
4. **Структурированность** - каждая страница отдельно, легко навигировать
5. **Markdown формат** - таблицы, списки, заголовки сохраняются

## Ограничения

1. **Форматы** - только PDF и DOCX (XLSX не поддерживается)
2. **Rate limit** - 1 запрос в секунду к VseGPT API
3. **Размер файла** - неизвестно точное ограничение (протестировано до 100KB)
4. **OCR качество** - зависит от качества PDF (отсканированные документы могут распознаваться хуже)
