# Результаты тестирования VseGPT Extract Text API

## Протестированные модели

### 1. utils/extract-text-1.0 (Базовая модель)

**Описание**: Универсальная модель для извлечения текста из документов.

**Поддерживаемые форматы**:
- ✅ **PDF** - работает отлично
- ✅ **DOCX** - работает отлично
- ❌ **XLSX** - НЕ поддерживается (500 Server Error)

**Качество извлечения**:
- Простой текст
- Таблицы представлены в текстовом виде (не markdown)
- Все цифры и данные извлекаются корректно
- Сохраняется структура документа

### 2. utils/pdf-ocr-1.0 (OCR модель)

**Описание**: Специализированная модель для PDF с оптическим распознаванием.

**Поддерживаемые форматы**:
- ✅ **PDF** - работает отлично

**Качество извлечения**:
- Markdown форматирование
- Таблицы в markdown формате (очень удобно!)
- Распознает и ссылается на изображения
- Дополнительные метаданные: DPI, размеры страниц, координаты изображений
- Опционально: возврат изображений в base64

**Дополнительные метаданные**:
```json
{
  "pages": [
    {
      "index": 0,
      "markdown": "...",
      "images": [...],
      "dimensions": {
        "dpi": 200,
        "height": 2339,
        "width": 1653
      }
    }
  ],
  "usage_info": {
    "pages_processed": 1,
    "doc_size_bytes": 94029
  }
}
```

## Выводы

### Для интеграции в MCP сервер рекомендуется:

1. **Использовать обе модели** с автоматическим выбором:
   - PDF → `pdf-ocr-1.0` (лучше для таблиц и сложных документов)
   - DOCX → `extract-text-1.0`

2. **Не поддерживать XLSX** напрямую
   - Альтернатива: попросить пользователя сохранить как PDF или использовать другой инструмент

3. **Ограничения API**:
   - Rate limit: не более 1 запроса в секунду
   - Необходимо добавить задержки между запросами

4. **Обработка кириллицы в именах файлов**:
   - API не принимает кириллицу в `filename` параметре
   - Решение: транслитерация или использование упрощенных имен

## Примеры использования

### PDF (лабораторные результаты)
```python
result = extract_text("IFA.pdf", model="utils/pdf-ocr-1.0")
# Получаем красиво оформленный markdown с таблицами
```

### DOCX (договоры, отчеты)
```python
result = extract_text("contract.docx", model="utils/extract-text-1.0")
# Получаем полный текст документа
```

**Оценка**: ⭐⭐⭐⭐⭐ (5/5)

- Все данные извлечены корректно (титры, номера образцов, даты)
- Таблицы представлены структурированно
- Метаданные сохранены (лаборатория, дата анализа, тип теста)
- Подходит для автоматической обработки результатов ПЦР, ИФА, бактериологических посевов

## Рекомендации по интеграции

1. Создать единый MCP инструмент `extract_document` с параметрами:
   - `file_path` или `file_base64` - путь к файлу или base64
   - `auto_detect_model` (default: true) - автоматический выбор модели
   - `model` (optional) - принудительный выбор модели
   - `return_images` (optional, для pdf-ocr) - возврат изображений

2. Добавить rate limiting (1 req/sec)

3. Обработка ошибок:
   - 500 Server Error → "Формат не поддерживается"
   - Rate limit → автоматический retry с задержкой
   - Валидация форматов на клиенте
