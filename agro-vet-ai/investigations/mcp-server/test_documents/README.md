# Тестирование извлечения текста из документов

Эта директория содержит тестовые файлы для проверки VseGPT API извлечения текста.

## Необходимые тестовые файлы

Поместите в эту директорию примеры следующих типов файлов:

### 1. PDF файл с лабораторными результатами
- **Название**: `lab_result_example.pdf`
- **Содержимое**: Типичный отчет из ветеринарной лаборатории
- **Цель**: Проверить качество извлечения текста из PDF

### 2. DOCX файл с лабораторными результатами
- **Название**: `lab_result_example.docx`
- **Содержимое**: Результаты анализов в формате Word
- **Цель**: Проверить работу с DOCX форматом

### 3. XLSX файл (экспериментально)
- **Название**: `lab_result_example.xlsx`
- **Содержимое**: Табличные данные с результатами
- **Цель**: Проверить, может ли API обрабатывать Excel файлы

### 4. PDF с изображениями/таблицами (для OCR)
- **Название**: `lab_result_scan.pdf`
- **Содержимое**: Отсканированный документ с таблицами/графиками
- **Цель**: Проверить OCR возможности

## Как запустить тесты

### Установка зависимостей

```bash
pip install requests python-dotenv
```

### Базовый тест (extract-text модель)

```bash
# PDF файл
python test_document_extraction.py test_documents/lab_result_example.pdf

# DOCX файл
python test_document_extraction.py test_documents/lab_result_example.docx

# XLSX файл (экспериментально)
python test_document_extraction.py test_documents/lab_result_example.xlsx
```

### Тест с OCR (для отсканированных PDF)

```bash
python test_document_extraction.py test_documents/lab_result_scan.pdf --model pdf-ocr
```

### Тест с сохранением изображений

```bash
python test_document_extraction.py test_documents/lab_result_scan.pdf \
  --model pdf-ocr \
  --with-images \
  --output result.json
```

## Что проверяем

1. **Качество извлечения текста**
   - Полнота извлеченного текста
   - Сохранение структуры (таблицы, списки)
   - Корректность распознавания цифр и единиц измерения

2. **Форматирование (Markdown)**
   - Сохраняется ли структура документа
   - Как представляются таблицы
   - Как выделяются заголовки

3. **Метаданные**
   - Какая дополнительная информация возвращается API
   - Разбиение по страницам
   - Информация о картинках

4. **Производительность**
   - Время обработки
   - Ограничения по размеру файла

## Ожидаемые результаты

После тестирования мы должны понять:
- Какие форматы поддерживаются лучше всего
- Насколько качественно извлекается текст из таблиц
- Нужен ли OCR для обработки типичных лабораторных результатов
- Какие ограничения есть у API

## Примечания

- API ключ берется из `.env` файла (переменная `OPENAI_API_KEY`)
- Для работы API необходимо отключить прокси (скрипт делает это автоматически)
- Максимальный размер файла и другие ограничения будут определены в процессе тестирования
